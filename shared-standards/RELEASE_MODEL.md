# Release Model

## Release Philosophy

Use milestone-based releases.

## Version Targets

- v0.1 → repository structure established
- v0.3 → early lessons operational
- v0.5 → partial course delivery capable
- v0.9 → complete draft
- v1.0 → production release

## Status Definitions

- Draft
- Alpha
- Beta
- Release Candidate
- Production
- Archived
