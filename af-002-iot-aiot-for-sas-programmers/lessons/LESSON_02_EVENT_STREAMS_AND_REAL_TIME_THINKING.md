# Lesson Purpose

Move from single telemetry records to operational event sequences.

---

# Learning Objectives

- recognize sequential telemetry
- identify changes over time
- understand events as operational signals
- connect event thinking to SAS analysis

---

## Activity

Use the sample telemetry CSV and identify simple changes over time.

Look for:

- changes in brightness
- timestamps
- threshold crossings
- repeated readings

## Recap

Real-time systems are built from connected operational events.
