# AF-002 Governance

This folder contains governance documents specific to:

AF-002 — IoT/AIoT for SAS Programmers

Purpose:
Provide operational governance guidance for:
- instructional runtime behavior
- evidence handling
- survivability practices
- Human-in-Command alignment
- deployment consistency
- operational safety

Governance at this level is intentionally:
- lightweight
- operational
- practical
- runtime-informed

These documents are NOT intended to create:
- enterprise bureaucracy
- heavy compliance structures
- theoretical governance frameworks

Instead, they exist to preserve:
- operational trust
- instructional consistency
- learner safety
- reproducibility
- survivability

---

# Current Governance Documents

## AF002-GOV-001_Controlled_Evidence_Doctrine.md

Purpose:
Establishes operational controls for instructional evidence used during runtime walkthroughs and operational lessons.

Key principle:

Operational evidence used during instructional runtime must be:
- approved
- reviewable
- reproducible
- stable across sessions

This doctrine specifically prohibits:
- dynamically generated runtime imagery
- uncontrolled visual evidence artifacts
during operational instructional sessions.

# Governance Scope

Governance in AF-002 occurs at the:
- course level
- lesson level
- workflow level
- operational runtime level

This allows governance to remain:
- proportional
- practical
- context-aware

Some AgentForge tools or courses may intentionally allow:
- image generation
- synthetic media creation
- VisualBrief generation

Those workflows should define governance appropriate to their own operational requirements.
