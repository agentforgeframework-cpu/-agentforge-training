# Observation

Runtime-generated imagery repeatedly damaged:
- evidence continuity
- learner trust
- operational realism
- instructional survivability

Prepared and approved artifacts dramatically improved runtime stability.

---

# Runtime Failures Observed

Generated images introduced:
- visual inconsistency
- telemetry mismatch
- operational confusion
- learner distrust

These failures occurred even when the generated images themselves appeared visually acceptable.

The issue was:
operational evidence continuity.

---

# Important Discovery

Operational lessons involving telemetry interpretation increasingly behave like:

```text
controlled evidence environments
```

rather than:
```text
traditional educational illustration.
```

This changes governance requirements significantly.

---

# Runtime Governance Conclusion

Operational instructional sessions should use ONLY:
- approved images
- approved screenshots
- approved overlays
- approved telemetry artifacts

Runtime-generated imagery should NOT be used during operational lessons.

---

# Strategic Result

This runtime finding directly resulted in:

AF002-GOV-001
Controlled Evidence Doctrine

---

# Human-in-Command Alignment

Controlled evidence improves:
- reproducibility
- operational trust
- evidence verification
- learner skepticism
- telemetry interpretation discipline

---

# Important Clarification

This governance finding applies specifically to:
- operational instructional runtime
- evidence interpretation workflows

It does NOT prohibit:
- VisualBrief workflows
- image generation tooling
- synthetic media experimentation
- approved media development pipelines

Governance remains:
course-specific and workflow-specific.

---

# Operational Conclusion

Operational realism depends on evidence discipline.
