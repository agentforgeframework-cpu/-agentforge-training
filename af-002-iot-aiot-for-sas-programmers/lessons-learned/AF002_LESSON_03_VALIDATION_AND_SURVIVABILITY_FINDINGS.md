# Major Instructional Discovery

A major educational discovery emerged:

```text
Operational understanding emerges through guided operational interaction.
```

Operational cognition emerged most effectively when learners:
- observed telemetry
- noticed anomalies
- formed operational hypotheses
- interpreted results
- THEN received explanation

This validated:

```text
Observe
→ Interpret
→ Explain
```

as a foundational instructional pattern.

# Operational Accessibility Principle

A major instructional principle emerged:

## AF-002 Operational Accessibility Principle

AF-002 is intentionally designed so learners can:
- understand the workflow themselves
- run the workflow themselves
- modify the workflow themselves
- restart the workflow themselves
- explain the workflow themselves

using ordinary personal or workplace computing environments.

The course intentionally avoids unnecessary dependence on:
- hyperscale infrastructure
- enterprise orchestration systems
- inaccessible deployment environments
- distributed operational complexity

This does NOT prohibit:
- enterprise deployment
- cloud deployment
- operational scaling

Instead it preserves:

```text
a motivated learner should plausibly be able to
experiment,
learn,
and understand the operational workflow independently
```

This principle strongly aligned with the validated AF-002 identity.
